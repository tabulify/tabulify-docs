===== Note =====


The reference can found on the [[https://www.postgresql.org/docs/current/datatype.html|Postgres Data Type page]].

==== Clob ====

The sql ''CLOB'' data type is not supported by [[postgres|PostgreSql]]. (ie the CLOB feature (''T041-02'') is in [[https://www.postgresql.org/docs/current/unsupported-features-sql-standard.html|the unsupported Features]]).

''Tabulify'' transform then:
  * a ''clob'' automatically in a [[https://www.postgresql.org/docs/current/datatype-character.html|text]] postgres data type
  * a ''text'' postgres column in a ''clob'' data type for external database.

==== Float ====

The SQL variable type ''float'' (ie ''float(p)'') does not exists.

We map it to:
    * ''real'' for a single precision (ie ''float4'')
    * ''double'' for a double precision (ie ''float8'')

==== Tinyint ====

''tinyint'' does not exist. We map it to ''smallint''


==== Unicode character (Nchar and NVarchar) ====
The unicode character data type ''Nchar'' and ''NVarchar'' does not exist in Postgres (([[https://www.postgresql.org/docs/current/datatype-character.html|Postgres Character Datatype]])).

If you want to transfer Unicode character to Postgres, you need to:
  * have a database with an [[https://www.postgresql.org/docs/current/multibyte.html|Unicode character set]]
  * and use the following respective data type ''Char'' and ''Varchar''

If Tabulify need to create a table, it will creates  ''Char'' and ''Varchar'' columns.

==== Decimal ====

Note that if you use ''decimal'' in your ''create table'' statement, the type column created will be a ''numeric'' in Postgres.

It's just a naming difference because the two type are equivalent but if you want to make a [[:docs:op:diff|schema diff]], you will see a name difference.

That's why when migrating from another database, the ''decimal'' data type is mapped to a ''numeric''.


==== Postgres Extra Types ====

=== Info-schema datatypes ===

We support the [[https://www.postgresql.org/docs/current/infoschema-datatypes.html|infoschema-datatypes]] so that you query any columns in the [[https://www.postgresql.org/docs/current/information-schema.html|information schema]].


=== Network Address Types ===

We support the [[https://www.postgresql.org/docs/current/datatype-net-types.html|Network address type]].




=== UUID ===

We support the [[https://www.postgresql.org/docs/current/datatype-uuid.html|UUID type]].

=== Name ===

We support the [[https://www.postgresql.org/docs/current/datatype-character.html#DATATYPE-CHARACTER-SPECIAL-TABLE|special name type]] (63, varying character)



==== Unsigned Integer does not exists in Postgres ====


Note that unsigned integer (ie positive integer only) does not exists in Postgres.

See this [[https://www.postgresql.org/message-id/3180774.1733588677%40sss.pgh.pa.us|thread for why]]
